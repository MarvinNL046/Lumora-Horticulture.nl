## ðŸ“ Writing Guidelines

### Tone & Style
- **Professional but Approachable**: Technical expertise without jargon overload
- **Data-Driven**: Use specific numbers, percentages, measurements
- **Actionable**: Clear recommendations, not just information
- **Honest**: Acknowledge trade-offs, no overselling

### Content Structure
1. **Hook**: Start with clear value proposition
2. **Context**: Why this matters to the reader
3. **Details**: Technical specifications and comparisons
4. **Guidance**: Help reader make decision
5. **Action**: Clear CTA with next steps

### Technical Writing Best Practices
- Define technical terms on first use
- Use tables for data-heavy comparisons
- Break up text with subheadings every 2-3 paragraphs
- Include visual aids (icons, emojis) for scannability
- Provide examples and real-world applications
